# 空间分辨本征模图方案（待用户确认，不直接运行）

## 触发条件

table-only ringdown 已经给出清晰的主候选峰：

| run | 主信号 | 主峰 | 状态 |
|---|---|---:|---|
| `Bz` | `m_z`, `E_total` | `173.66 GHz` | 最强候选本征峰 |
| `Bx` | `m_x/m_y` | 无清晰离散峰 | 不建议优先做空间模图 |

因此空间模图第一优先级应放在 `Bz` 激励下的 `173.66 GHz` 峰。若用户希望探索弱峰，可追加 `38.82 GHz` 或 `126.67 GHz`，但这些峰功率远低于主峰，论文优先级较低。

## 推荐方案

| 参数 | 建议值 | 理由 |
|---|---:|---|
| 初始态 | `centered_Ku10k.out/m000020.ovf` | 与 ringdown 和现有频扫同源 |
| 激励 | `Bz` weak sinc pulse, `B0=5 mT`, `fc=2000 GHz` | 与已完成本征谱严格一致 |
| 保存对象 | 首选完整 `m` vector OVF；若 Mumax3 vet 支持，再考虑只存 `m_z`/`m_x`/`m_y` 分量 | 空间 FFT 需要逐胞时间序列 |
| 总时长 | `0.3 ns` 起步，必要时 `0.5 ns` | 0.3ns 分辨率约 3.3GHz，足以围绕 173.66GHz 做模图；0.5ns 对应 2GHz |
| OVF 保存间隔 | `0.2 ps` | Nyquist 2.5THz，覆盖 173.66GHz 和 1100GHz；比 0.05ps 减少 4 倍文件数 |
| table 保存间隔 | `0.05 ps` | 保留与块2一致的平均谱诊断 |
| 输出 | `mode_map_Bz_173GHz.out/` | 单独目录，避免污染 table-only run |
| 分析 | 逐胞 FFT，提取 173.66GHz 附近幅度 `|FFT[m_z]|`，并可同步看 `m_x/m_y` twisting 分量 | 对应 Raftrey-Fischer Fig.3 的空间局域判据 |

## 开销估计

粗略估算按 `100^3` cells、每帧三分量 float32 计算：

| 配置 | 帧数 | 单帧原始数据 | 单 run 数据量 | 备注 |
|---|---:|---:|---:|---|
| `0.3 ns`, `0.2 ps` | 1500 | 约 12 MB | 约 18 GB | 推荐起步 |
| `0.5 ns`, `0.2 ps` | 2500 | 约 12 MB | 约 30 GB | 更好频率分辨率 |
| `0.5 ns`, `0.05 ps` | 10000 | 约 12 MB | 约 120 GB | 不推荐，除非只存分量且磁盘充足 |

分析内存也不能一次性把所有帧堆进数组。后续脚本应采用 slab/chunk streaming FFT，或者只对核心附近 ROI 做 FFT。若直接读全量 `100^3 x 1500 x 3`，内存会达到几十 GB，不适合轻率运行。
